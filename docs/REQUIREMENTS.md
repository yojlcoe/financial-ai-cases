# 事例調査AIエージェント - 要件定義書

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| システム名 | 事例調査AIエージェント（Case Study Research Agent） |
| 目的 | 金融機関のAI・DX事例を自動収集・要約・分類し、定期的にレポートを生成する |
| 対象ユーザー | 社内リサーチャー、戦略企画部門 |

---

## 2. 機能要件

### 2.1 コア機能

| 機能ID | 機能名 | 説明 |
|--------|--------|------|
| F-001 | 定期実行 | 日次または週次でバッチ実行 |
| F-002 | 企業別順次検索 | 設定された企業リストを順番に処理 |
| F-003 | 期間指定検索 | `after:YYYY-MM-DD before:YYYY-MM-DD`形式で期間指定 |
| F-004 | 公式サイト直接アクセス | 企業のプレスリリースページを直接クローリング |
| F-005 | 記事内容抽出 | タイトル・本文・発行日・URLを取得 |
| F-006 | 期間内検証 | 記事の発行日が指定期間内か検証 |
| F-007 | AI要約・分類 | Gemma 3 4Bで要約生成、カテゴリ・タグ付与 |
| F-008 | 複数記事統合 | 同一トピックの記事を統合し、複数ソース記載 |
| F-009 | レポート生成 | 見出し付きの整理されたレポート出力 |

### 2.2 管理機能

| 機能ID | 機能名 | 説明 |
|--------|--------|------|
| M-001 | 企業管理 | 対象企業の追加・編集・削除 |
| M-002 | URL管理 | 企業別の対象URLを設定 |
| M-003 | 期間設定 | 検索期間の設定 |
| M-004 | スケジュール設定 | 実行頻度（日次/週次）の設定 |
| M-005 | 実行履歴 | 過去の実行結果を閲覧 |

---

## 3. 非機能要件

| カテゴリ | 要件 |
|----------|------|
| 正確性 | 情報がない場合は「該当なし」と明記。想像・捏造禁止 |
| トレーサビリティ | 全ての情報にソースURLを必須記載 |
| コスト | ローカルLLM使用で推論コスト0円 |
| 拡張性 | 企業・URLの追加が容易 |
| 運用性 | Dockerコンテナでポータブルに運用 |
